P00003 Hooks Guide (클라이언트에서 서버 액션 직접 호출 구조)

대상 파일:
- page.tsx
- actions.ts (서버 액션 파일, 훅 없음)

==================================================
1) useState
==================================================

[라인 7] const [serverKeyword, setServerKeyword] = useState("");
- 역할: 서버 액션 호출 시 전달할 검색어 상태.
- 사용 이유: 서버 조회 파라미터를 input과 동기화해야 하므로 로컬 상태가 필요.
- 언제 쓰나: 입력값, 토글, 선택값처럼 즉시 UI 반영이 필요한 기본 상태 관리.

[라인 8] const [clientFilter, setClientFilter] = useState("");
- 역할: 이미 조회된 rows를 클라이언트에서 추가 필터링할 때의 입력값 상태.
- 사용 이유: 서버 재요청 없이 로컬에서 빠르게 필터링하려는 목적.

[라인 9] const [rows, setRows] = useState<SearchRow[]>([]);
- 역할: 서버 액션 결과 데이터 저장.
- 사용 이유: 서버에서 받은 배열을 렌더링 원본으로 보관.

[라인 10] const [message, setMessage] = useState(...);
- 역할: "조회 중", "완료" 같은 진행 메시지 관리.
- 사용 이유: 비동기 UX 피드백 제공.

정리:
- useState는 이 페이지에서 "입력값", "서버 응답 데이터", "상태 메시지"를 분리해 관리하는 핵심 훅.

==================================================
2) useTransition
==================================================

[라인 11] const [isPending, startTransition] = useTransition();
- isPending: transition 업데이트가 처리 중인지 여부.
- startTransition: 내부 setState를 비긴급 업데이트로 처리.

[라인 23-31] onSearch 내부 startTransition(async () => { ... })
- 서버 액션 호출 후 setRows, setMessage 업데이트를 transition으로 묶음.
- 의미:
  - 입력 반응성(타이핑/클릭)을 우선 유지하고
  - 결과 반영 렌더링은 비긴급으로 처리.

언제 쓰나:
- 대량 리스트/차트처럼 응답 후 렌더링 부담이 있는 화면.
- 네트워크 속도 개선용이 아니라, 렌더 우선순위 조절용.

==================================================
3) useDeferredValue
==================================================

[라인 14] const deferredFilter = useDeferredValue(clientFilter);
- 역할: clientFilter의 즉시값 대신 "지연된 값"을 만들어 무거운 계산에 사용.
- 목적: 입력창 타이핑은 즉시 반응하고, 필터 계산은 한 템포 늦게 수행.

[라인 15] const isStale = clientFilter !== deferredFilter;
- 역할: 현재 입력값과 지연값 차이를 통해 "계산 중/최신 반영" 상태 표시.
- UX 의미: 사용자는 지금 필터 계산이 따라오는 중인지 확인 가능.

언제 쓰나:
- 검색창 입력에 따라 큰 리스트 필터링/정렬/변환이 반복될 때.
- 입력 끊김을 줄이고, 체감 성능을 부드럽게 만들고 싶을 때.

==================================================
4) useMemo
==================================================

[라인 17-21] const filteredRows = useMemo(...)
- 의존성: [rows, deferredFilter]
- 역할: 지연된 필터 값 기준으로 필터 결과를 메모이제이션.
- 효과:
  - rows/deferredFilter가 바뀔 때만 재계산
  - 불필요한 반복 계산 감소

언제 쓰나:
- 계산 비용이 있는 파생 데이터(필터/정렬/집계)를 렌더마다 다시 만들고 싶지 않을 때.

==================================================
5) 훅 조합 흐름 요약
==================================================

1. useState(serverKeyword)로 서버 검색어 입력 관리
2. 조회 버튼 클릭 -> useTransition(startTransition) 안에서 서버 액션 호출
3. 응답 데이터는 useState(rows)에 저장
4. useState(clientFilter) 입력은 즉시 반영
5. useDeferredValue(clientFilter)로 필터 계산 타이밍을 늦춤
6. useMemo(filteredRows)로 지연값 기준 필터 결과 계산
7. isPending / isStale로 진행 상태를 UI에 표시

==================================================
6) actions.ts 참고
==================================================

- actions.ts는 서버 액션 파일이므로 React 훅을 쓰지 않음.
- 역할은 데이터 생성/조회/검증 같은 서버 로직 처리.
- 훅은 page.tsx(클라이언트 컴포넌트)에서 UI 상태/렌더 제어에 사용.
