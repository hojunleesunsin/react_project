P00008 Hooks Line-by-Line Guide

대상 파일:
- page.tsx
- actions.ts (참고: 훅은 없음, 서버 액션 로직만 존재)

----------------------------------------
[page.tsx 훅 설명]
----------------------------------------

1) L3: import { useState, useTransition } from "react";
- useState: 컴포넌트 내부의 로컬 상태를 저장/갱신할 때 사용.
- useTransition: 상태 업데이트 우선순위를 낮춰(비긴급 처리) UI 끊김을 줄일 때 사용.

2) L7: const [title, setTitle] = useState("");
- 제목 input의 현재 값을 관리하는 상태.
- 사용 시점: 사용자가 입력할 때마다 값이 즉시 반영되어야 하는 "긴급 UI" 상태.

3) L8: const [description, setDescription] = useState("");
- 설명 textarea의 현재 값을 관리하는 상태.
- 사용 시점: 폼 입력값처럼 컴포넌트가 직접 제어하는 controlled input에서 사용.

4) L9: const [submitCount, setSubmitCount] = useState(0);
- 저장 버튼 클릭 후 성공/실패와 별개로 "시도 횟수"를 표시하는 상태.
- 사용 시점: 사용자 행동 횟수, 클릭 횟수, 탭 전환 횟수 같은 단순 카운트에 적합.

5) L10-L15: const [result, setResult] = useState<SaveNoteResult>(...)
- 서버 액션 응답(성공 여부, 메시지, 에러)을 한 번에 담는 상태.
- 사용 시점: 여러 값(ok/message/errors)을 분리하지 않고 "응답 객체 단위"로 관리할 때 유리.

6) L16: const [isPending, startTransition] = useTransition();
- isPending: transition 처리 중인지 여부.
- startTransition: 내부 setState를 비긴급 업데이트로 처리.
- 사용 시점:
  - 응답 이후 렌더링 비용이 큰 화면(차트/리스트 재렌더)이 있을 때.
  - 입력 반응성은 유지하고, 결과 반영은 잠깐 늦춰도 되는 경우.

7) L18-L24: onClickSave 내부 startTransition(async () => { ... })
- L19: startTransition 시작.
- L20: 클라이언트에서 서버 액션(saveNoteAction) 직접 호출.
- L21: setSubmitCount로 시도 횟수 증가.
- L22: setResult로 서버 응답 반영.
- 포인트:
  - 네트워크 자체를 빠르게 만드는 것이 아니라, 응답 후 렌더 우선순위를 조절.
  - 화면이 가벼우면 transition 체감이 작을 수 있음.

8) L41-L43, L56-L58: onChange에서 setTitle / setDescription
- controlled component 패턴.
- 사용 시점: 입력값 검증, 실시간 포맷팅, submit 시 최신 값 보장 필요할 때.

9) L69, L74: disabled={isPending}, 저장 중 텍스트
- isPending으로 사용자 중복 클릭 방지 + 진행 상태 표시.
- 사용 시점: 비동기 작업 중 UX 피드백이 필요할 때.

10) L46, L61-L63, L82-L84
- result.errors / result.message로 서버 검증 결과를 UI에 렌더링.
- useState(result) 패턴을 통해 "서버 결과 -> 화면" 흐름을 단순하게 유지.

----------------------------------------
[actions.ts 참고]
----------------------------------------

- actions.ts에는 React 훅이 없다.
- 이유: 서버 액션 파일은 React 렌더링 컨텍스트가 아니며, 훅 사용 대상이 아니다.
- 역할: 입력 검증, 비즈니스 로직 실행, 결과 객체 반환.
- page.tsx의 훅(useState/useTransition)은 이 결과를 받아 사용자 화면을 제어한다.

----------------------------------------
[언제 어떤 훅을 쓰는가? (P00008 기준)]
----------------------------------------

- useState
  - 입력값, 카운터, 서버 응답처럼 "현재 화면 상태"를 저장할 때 기본 선택.

- useTransition
  - 큰 렌더링 업데이트를 비긴급으로 내려 사용자 입력 반응성을 지키고 싶을 때.
  - 필수 훅은 아니며, 화면이 단순하면 useState만으로도 충분.

- useActionState (이 파일에서는 미사용)
  - 주로 form action 패턴과 함께 서버 액션 결과 상태를 자동 관리할 때 적합.
  - 현재 P00008은 "클라이언트에서 직접 await 호출" 패턴이므로 useActionState를 쓰지 않음.
